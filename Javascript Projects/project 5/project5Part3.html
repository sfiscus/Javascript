Problem Statement

Begin by executing the program already present in the Part 3 JavaScript library file, a program that currently accesses the Item Inventory Records and lists out records with item stock amounts of 100 or more.

Then, accomplish maintenance to the program.

After the modification, the program must still output rows within the HTML table, a table with rows consisting of the item number, the item description, and the item stock amount.

But after modification, the table must have rows only for items that had an item stock amount that fell within the range specified by the user (more on the range below). And at the end of the table, the program must still have the two total lines, but now showing the number of items that had stock amounts within the specified range, as well as the total number of records found in the file.

As mentioned above, the modified program must now no longer be hard-coded to look for items with stock amounts of 100 or more. Rather, the program must now ask the viewer to enter in a lower range boundary, and, an upper range boundary. This acquiring of the two boundary numbers from the viewer must be done via either an HTML form, or via use of the prompt method. Then, the modified program must find all the records in the file with item stock amounts that are equal to or greater than the lower range boundary for stock amount, and, less than or equal to the upper range boundary for stock amount.

Before using the viewer-entered range boundaries to process against the Item Inventory file, the modified program must first validate the two entered range boundary numbers as follows:  the first number entered by the user (the lower range boundary) must be numeric, and greater than or equal to zero. The second number entered by the user (the upper range boundary) must be numeric, AND, it must be greater than or equal to the first number entered.

If any of these validation requirements is not met, the program MUST put out an error message(s) warning the user of incorrect inputs. (In such a case, preferably the program would then stop, and proceed no futher. However, you can allow the program to just proceed, provided that the user has been warned to ignore the output, and, has been told to start over by refreshing the Web page. Another alternative would be to have the program loop until they DO put in correct inputs.)

The heading above the output table MUST also be modified so that it no longer says 'Item Inventory Records With Stock Amounts 100 or More' but rather includes the user-entered upper and lower range boundary numbers. See the suggested wording in the example for this heading (but with the lower and upper range boundaries reflecting the numbers the user input for any given run, of course). The exact wording of the heading is your choice, so long as the upper and lower range boundary numbers input by the user are included within it.
