Problem Statement

Write a program that will find all the unique zip codes in the Zip Code Study Records. The program must access the record set and extract the unique zip codes and store them. Then output the unique zip codes from storage and display them as an ordered or unordered list of the unique codes (on the Web page). Note: This must not be just a comma-delimited "dump" of the codes, but rather it must use the 'ol' or 'ul' HTML elements.   The program must use modules for all identifiable tasks.

NOTE: The program may not have any zip code values hard-coded within the program, as was permitted in Learning Unit 4. This LU5 program has to function correctly without knowing in advance how many unique zip codes exist in the record set, nor what the values of these unique zip codes are. Rather, the program must make one pass through the records in the file, using array storage to collect zip code values. Any arrays used in this program must be empty at the time the program begins to run.

NOTE: The term 'unique' can mean different things in English depending on the context. Make sure you understand what the phrase 'unique zip codes' means for this application.  Example: The following set of zip codes (53703, 53702, 53708, 53702) has three unique zip codes.  Example: The followiing set of zip codes (53711, 53711, 53711, 53705, 53705) has two unique zip codes.   Example: The following set of zip codes (53708, 53708, 53708) has one unique zip code.


NOTE: The program must be set up using at least two other modules (i.e. sub-functions) in addition to the mainline controller module (function). Remember to follow the course standards on functions.


  • Project Notes:
    • Use the Zip Code Study Records for the data.  NOTE: The sample program illustrating accessing the Zip Code Study Records contains an IF statement that stops the looping after only 25 records have been read. That IF is merely illustrative of how one could temporarily work with a small subset of a large file, for purposes of initial testing. Naturally, that IF would later be removed before placing a program into production. Thus, this Part 1 Project program must work with all of the records in the file, and not just the first 25 records.
