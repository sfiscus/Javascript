Problem Statement

Begin by executing the program already present in the Part 4 JavaScript library file, a program that currently asks the user to enter any number of strings, one prompt() at a time, until the user keys in the string "quit".

Note that the program will only accept an odd number of strings, but there is no upper limit on the number of strings it will accept. Nor are their any restrictions on what characters the user can key in for each string.

Then, modify the program.   After the modification, the program must still do everything it currently does, but now it also must output the "middle" value keyed in by the user. That is, the program must not only display the entered strings (just as is currently does), and number of strings entered (just as it currently does), but it must now also put out the "middle" value entered. And it must place this value into the heading line (which is already present in the program).  Note: the word "quit" does NOT count as one of the entered strings.

Note that the required new output is NOT the median of the values entered, but rather is simply the "middle" value entered. E.g. if the user keyed in say:

   •bottle
   •catch
   •zweeter
   •bat
   •zebras

then the "middle" valued keyed would be the content of the third slot of the array, namely "zweeter".

The program must continue to have no other inputs than just the set of strings keyed in by the user (and the "quit" string of course). And it must continue to have the ability to work with any (odd) number of strings.

  • Project Notes:
    • Make no program modifications without first having done a complete plan for what you will be doing.
    • Read about the  Math.floor()  function as it applies to a mathematical result that has a decimal portion. This could be very handy for this project Part 4.
    • Proceed with modifying the program using 'versions'. Test each version to make sure it is accomplishing what you planned for it to accomplish, before going ahead with the next version. As is often the case, the  alert()  method will be handy for verifying that a version is working properly.
    • In your test cases, be sure to include one in which the user has entered say three strings, as well as one in which the user has entered say 17 or some other larger number of strings.
